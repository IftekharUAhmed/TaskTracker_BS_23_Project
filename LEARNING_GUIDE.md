# Learning Guide


## Source Priority


When implementing any feature:


Priority 1:

Project Requirement PDF


Priority 2:

My instructor provided me some modules (Module 1-20)



Priority 3:

Industry best practice / external solution



## Coding Style Rule


If a concept exists in instructor modules:

Follow the same approach and style.


Example:

If Repository Pattern exists in modules:

Use that implementation style.


If Dependency Injection exists in modules:

Follow the same DI approach.


If DTO pattern exists:

Prefer that pattern.



If a required project feature is not covered in modules:

Then use:
- official documentation
- industry standard approach
- external research



## Learning Goal

Do not only complete the project.

Maintain consistency with what I learned from modules and also want to make a best , scalable , well achitectured project.
